## 摘要
大型语言模型（LLM）越来越多地被用作进化机器人设计的建议生成器，然而大多数循环仍是无记忆的：模拟器结果塑造下一代种群，但并未作为可重复使用的设计知识加以保存。我们提出了Auto-Robotist，这是一种自我进化的LLM智能体，它将形态搜索轨迹提炼为显式的自然语言技能库。每个技能存储一个结构原型、基于证据的正向和负向规则，以及支持这些规则的已评估设计，使得设计记忆可检查而非隐含在种群中。在搜索过程中，智能体检索技能以调节LLM对精英体的编辑，同时保留遗传算法（GA）突变路径以进行探索；在评估后，它通过添加、诊断和合并操作更新技能库。在涵盖运动、穿越和物体交互的七个EvoGym任务中，Auto-Robotist改善了冷启动5x5搜索，并将学到的技能迁移到10x10设计空间，其中基于参考条件的迁移在每个任务上都优于GA。这些结果表明，LLM智能体可以将昂贵的物理评估转化为可重复使用、可审计的设计原则。我们的代码将在接收后发布。

## Abstract
Large language models (LLMs) are increasingly used as proposal generators for evolutionary robot design, yet most loops remain memoryless: simulator results shape the next population but are not preserved as reusable design knowledge. We present Auto-Robotist, a self-evolving LLM agent that distills morphology-search traces into an explicit natural-language skill library. Each skill stores a structural archetype, evidence-grounded positive and negative rules, and the evaluated designs that support them, making design memory inspectable rather than implicit in a population. During search, the agent retrieves skills to condition LLM edits of elite bodies while retaining a Genetic Algorithm (GA) mutation path for exploration; after evaluation, it updates the library through Add, Diagnose, and Merge. Across seven EvoGym tasks spanning locomotion, traversal, and object interaction, Auto-Robotist improves cold-start 5x5 search and transfers learned skills to 10x10 design spaces, where reference-conditioned transfer outperforms GA on every task. These results suggest that LLM agents can convert expensive physical evaluations into reusable, auditable design principles. Our code will be released upon acceptance.

## 2. 方法论：核心思想、关键技术细节

### 核心思想
提出 **Auto-Robotist**，一个自进化的LLM智能体，它将形态搜索轨迹（search traces）显式地提炼为**自然语言技能库**（natural-language skill library）。技能库存储结构原型（archetype）、基于证据的正/负规则，以及支撑它们的设计评估结果，使设计记忆可审查而非隐含在种群中。搜索时，智能体检索技能以引导LLM对精英体的编辑，同时保留遗传算法（GA）突变路径以维持探索；评估后，通过三个操作（Add, Diagnose, Merge）动态更新技能库。

### 关键技术细节

1. **技能表示（Skill Representation）**：每个技能分为三层：
   - **L1（结构原型）**：如 portal-frame（门式框架），带有适用条件。
   - **L2（规则）**：正向规则（提高适应度）和负向规则（失败模式），每个规则存储结构描述、支持证据以及运行中的平均亲本相对增益（mean parent-relative gain）。
   - **L3（观察）**：已评估的子代和亲本体、任务、尺度、适应度、增益、有效性状态和提案归因。该结构使记忆稳定且可审计。

2. **技能条件化提案（Skill-Conditioned Proposal）**：
   - 计算每个技能的平滑有用性估计 \( w_s = \frac{1 + \sum_{i=1}^{n_s} \text{clip}(g_{s,i}/\delta_{\max}, 0, 1)}{2 + n_s} \)，其中 \( g_{s,i} \) 是亲本相对增益，\( n_s \) 是观察数。
   - 检索后，按证据加权效用采样一个技能，提示LLM根据该技能编辑精英亲本，生成候选体。同时并行运行GA突变路径以确保覆盖。

3. **库维护（Library Maintenance）**：
   - **Add**：将未匹配现有技能的观察放入未分配证据池，每代LLM可从中创建新L1原型（避免每篇观察一个技能）。
   - **Diagnose**：根据新证据更新现有技能规则：使用指数移动平均更新规则平均增益；LLM决定规则是否支持、矛盾、创建或未定。负向规则作为一等公民存储（因为失败模式往往像成功一样可靠迁移）。
   - **Merge**：聚类由LLM生成的可能冗余的原型，仅合并描述相同结构原理的技能，保留证据追溯。

4. **跨尺度迁移（Cross-Scale Transfer）**：从源尺度（如5×5）迁移到目标尺度（如10×10）时，导入源技能库，保留L1和L2但丢弃原始观察，作为先验指导目标搜索。后续目标观察进入未分配池，可形成本机技能。无需体素上采样，因为技能规则基于功能部位关系（如“使用刚性侧框架稳定垂直致动器”），可跨分辨率迁移。

5. **算法流程**（Algorithm 1）：初始化种群和技能库；每代检索技能→LLM编辑精英→GA额外提案→有效性检查→PPO评估→路由观察→执行Add/Diagnose/Merge更新库。

## 3. 实验设计

### 使用数据集/场景
- **基准**：EvoGym（Bhatia et al., 2021），一个2D软机器人基准，任务性能依赖于支撑、接触、驱动和任务几何。
- **七个任务**：Walker（行走）、BridgeWalker（过桥）、Balancer（平衡）、Carrier（携带）、Climber（攀爬）、Jumper（跳跃）、Pusher（推动）。
- **两种设计空间**：5×5网格（冷启动，技能需从头学习）和10×10网格（迁移测试，源技能库导入）。

### Benchmark与对比方法
- **主要基线**：标准EvoGym遗传算法（GA），相同预算（5×5：Walker/BridgeWalker 250次评估，Balancer/Carrier/Climber 500次，Jumper/Pusher 750次；10×10：所有任务1000次）、种群大小25、生存率调度。
- **变体**：
  - `w/ ref`：迁移时导入5×5技能库并在提示中显示源精英体（参考设计）。
  - `w/o ref`：仅导入技能库，移除源身体示例，隔离规则迁移与视觉模仿。
- **负对照**：将5×5精英直接2×2上采样平铺到10×10（判断简单上采样是否有效）。

### 评估指标
- 最佳适应度曲线（best fitness so far）、最终最大适应度、收敛加速比（S）、领先占比（L）。

## 6. 主要结论与发现

1. **冷启动5×5搜索**：Auto-Robotist在全部7个任务上匹配或优于GA，平均收敛加速比1.47×，70%预算时间领先。Carrier、Jumper、Pusher上提升最大（如Pusher +1.42）。
2. **5×5 → 10×10迁移**：参考条件化Auto-Robotist在所有7个任务上超越GA，平均最终适应度提升+1.55，6个任务上领先占比≥90%。Carrier提升最显著（+5.71）。
3. **技能库比简单上采样更有效**：直接2×2上采样往往破坏功能（尤其Carrier、Pusher），而Auto-Robotist通过传递结构关系（如支撑框架、接触垫）实现更优的重新实例化。
4. **规则迁移优于复制原型**：仅技能迁移（移除参考体）在Walker、Carrier、Jumper上仍胜GA，表明技能编码了超越外观的功能信息；但BridgeWalker、Balancer、Climber需参考体锚定，表明抽象规则有时需要几何起点。
5. **库的可解释性**：学习的技能（如portal-frame、lower-bridge）包含正向和负向规则，暴露了反复出现的失败模式，使设计决策可审计。

## 7. 优点

- **问题新颖性**：将进化机器人设计重新定义为“将搜索经验转化为可迁移技能”，而非仅生成高适应度体。这是对LLM辅助设计领域的重要补充。
- **技能表示设计**：三层结构（原型+规则+观察）实现了抽象性、可迁移性和可审计性的平衡；负向规则作为一等公民纳入，符合工程常识。
- **记忆更新机制**：Add/Diagnose/Merge操作限制了无节制的技能膨胀，同时允许知识演化，避免了LLM参数更新成本。
